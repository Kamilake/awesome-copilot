# 보안

GitHub는 [GitHub](https://github.com/GitHub) 등 GitHub 조직에서 관리하는 모든 오픈소스 코드 저장소를 포함하여, 소프트웨어 제품 및 서비스의 보안을 중요하게 생각합니다.

[오픈소스 저장소는 버그 바운티 프로그램의 범위에 포함되지 않으므로](https://bounty.github.com/index.html#scope) 바운티 보상 대상은 아니지만, 발견된 사항이 적절한 관리자에게 전달되어 조치될 수 있도록 하겠습니다.

## 보안 이슈 신고

GitHub 소유 저장소에서 보안 취약점을 발견했다고 판단되는 경우, 협조적 공개(coordinated disclosure)를 통해 신고해 주시기 바랍니다.

**공개 GitHub issue, discussion, 또는 pull request를 통해 보안 취약점을 신고하지 마십시오.**

대신 opensource-security[@]github.com으로 이메일을 보내주시기 바랍니다.

문제를 더 잘 이해하고 해결할 수 있도록 아래 정보를 가능한 한 많이 포함해 주십시오:

  * 이슈 유형 (예: 버퍼 오버플로우, SQL 인젝션, 크로스 사이트 스크립팅)
  * 이슈가 발생하는 소스 파일의 전체 경로
  * 영향을 받는 소스 코드의 위치 (태그/브랜치/commit 또는 직접 URL)
  * 이슈를 재현하는 데 필요한 특수 설정
  * 이슈를 재현하기 위한 단계별 지침
  * 개념 증명(proof-of-concept) 또는 익스플로잇 코드 (가능한 경우)
  * 이슈의 영향 범위 및 공격자가 이를 악용할 수 있는 방법

이 정보는 신고를 더 신속하게 분류하는 데 도움이 됩니다.

## 정책

[GitHub의 Safe Harbor 정책](https://docs.github.com/en/site-policy/security-policies/github-bug-bounty-program-legal-safe-harbor#1-safe-harbor-terms)을 참조하십시오.
